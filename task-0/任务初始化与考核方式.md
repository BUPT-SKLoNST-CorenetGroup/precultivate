# 任务初始化和考核方式

## 1. 概述

task-0是后续task的准备工作。培训任务会依托于github进行，因此，git是一个必须要掌握的工具。通过本次任务，来学习如何将从github搜索需要的代码，以及如何将代码push到github上。

## 2. 任务描述

之后所有的培训任务将会通过github来发布，因此，第一个任务是，注册一个github的账户。

github注册成功后，查询仓库zxm-bupt/precultivate。后续任务发布会在该仓库发布。

> 建议给仓库一个star

在仓库中查询到任务后，完成仓库中的task，任务完成后，提交部分使用如下方式。

1. 创建一个自己的github仓库吗，并将其设置为public；
2. 将任务完成的报告和代码推送到github仓库；
3. 在zxm-bupt/precultivate提交一个任务完成的issue，在issue中附带自己的仓库地址。

> 你可以一个任务一个仓库，也可以所有任务共用一个仓库，仓库中目录的组织形式也完全由你来自主决定。实验报告的格式和文件形式也不设置限制，可以使用PDF，也可以word，当然还是建议尝试一下markdown。

## 3. 需要掌握的知识

1. git的基本命令init、push、add、commit的用法
2. github的基本使用方式
